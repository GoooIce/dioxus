# OHOS E2E 测试应用

这是一个专门为验证 Dioxus OHOS 平台支持而设计的端到端测试应用。

## 目的

这个应用包含了所有需要验证的核心 Dioxus 功能：

1. **WebView 渲染** - 验证内容正确显示
2. **用户交互** - 测试点击、输入、复选框、单选框
3. **状态管理** - 测试 use_signal、use_resource、context
4. **资源加载** - 验证 CSS、图片等资源加载
5. **列表渲染** - 测试动态列表和 key 属性

## 使用方法

### 1. 构建应用

```bash
# 开发构建
dx build --platform mobile --target ohos

# 发布构建
dx build --platform mobile --target ohos --release
```

### 2. 运行应用

```bash
# 启动开发服务器并部署到设备
dx serve --platform mobile --target ohos
```

### 3. 验证功能

使用 `E2E_TEST_CHECKLIST.md` 中的清单逐项验证应用功能。

## 测试场景

### 场景 1: WebView 渲染

- 应用启动后应该看到标题和说明文字
- 应该看到一个带渐变色的测试框
- 所有文本应该正确显示

### 场景 2: 用户交互

1. **点击测试** - 点击按钮，计数应该增加
2. **输入测试** - 在文本框输入内容，下方应该实时显示
3. **复选框** - 勾选/取消勾选，状态应该正确更新
4. **单选框** - 选择不同选项，应该只能选中一个

### 场景 3: 状态管理

1. **计数器** - 点击按钮，数字增加并自动重新渲染
2. **异步加载** - 启动后 1 秒应该显示"异步数据加载成功"
3. **Context** - 子组件应该显示从父组件传递的计数

### 场景 4: 资源加载

1. **图片** - 应该看到 emoji 图标（实际应用中应该是真实图片）
2. **CSS** - 应该看到三个不同颜色的框（主色、次色、成功）

### 场景 5: 列表渲染

1. **静态列表** - 应该显示 5 个项目
2. **动态添加** - 输入新项目名称并点击"添加"，列表应该更新

## 预期结果

### 正常运行时

- ✅ 所有文本正确显示
- ✅ 样式正确应用（颜色、间距、圆角等）
- ✅ 所有交互响应正常
- ✅ 没有崩溃或错误

### 可能的问题

如果出现问题，检查：

1. **空白屏幕** - WebView 可能未正确初始化
2. **样式缺失** - CSS 文件可能未正确加载
3. **交互无响应** - IPC 通信可能有问题
4. **崩溃** - 检查设备日志

## 调试

查看应用日志：

```bash
hdc shell hilog -T Dioxus
```

## 性能基准

这个应用的目标性能指标：

- **启动时间**: < 3 秒
- **内存使用**: < 50 MB
- **FPS**: 接近 60fps（滚动时）

## 扩展

要添加更多测试场景，在 `src/main.rs` 中添加新的组件函数，并在 `TestSections` 中引用它。

示例：

```rust
#[component]
fn MyNewTest() -> Element {
    rsx! {
        div { class: "test-section",
            h2 { "我的新测试" }
            // 测试内容
        }
    }
}
```

然后在 `TestSections` 中添加：

```rust
MyNewTest {}
```

## 清理

测试完成后，删除应用：

```bash
hdc uninstall com.dioxus.ohos.test
```
